JAVA简介

1.1 编程概述

1.1.1 计算机程序基础

计算机程序由一系列指令组成，这些指令依次执行，基本指令类型有：

  • 输入：程序获取数据，数据来源可以是文件、键盘、触摸屏、网络等。
  • 过程：程序对数据执行计算，例如将两个值相加，如 x + y。
  • 输出：程序将数据放入某个地方，例如文件、屏幕、网络等。

程序使用变量来引用数据，是因为当程序为变量（例如 x）赋新值时，变量的值也会随之改变。

1.1.2 算法概念

  • 算法是解决问题的方法和步骤，它关注的是逻辑。
  • 编程是把这些方法和步骤用计算机能理解的语言表达出来，它关注的是实现。

1.2 编程基础

1.2.1 第一个项目

  • 程序从 main() 方法开始，依次执行 main() 方法大括号 { } 内的语句。
  • 每个语句通常单独成行，以英文分号结尾。
  • 该int wage语句创建了一个名为 wage 的整型变量，wage = 20代表将 wage 赋值为 20。
  • print() 和 println() 方法会输出各种值。

1.2.2 基本输入

  • 程序通常会获取输入值，对这些输入值进行处理，然后将输出值显示在屏幕或其他位置。输入值通常来自键盘、文件、网页表单或应用程序中的字段等。

  • Scanner是一种文本解析器，可以从键盘等输入源获取数字、单词或短语。

1.2.3 基本输出

输出文本

  • 以下两个方法都是 Java 中用于控制台输出的基本方法，主要区别在于是否换行。

​ System.out.print() : 连续输出在同一行。
​ System.out.println() : 每个输出独占一行，可以单独使用输出空行。

  • 输出文本的方式如下：System.out.print(“desired text”); 双引号内的文本称为字符串字面量。

输出变量值

使用同一条语句输出多个内容

  • 程序员通常通过将文本、变量值和新行的输出合并，为每一行输出使用一条输出语句，只需用+符号分隔各项即可。

  • 这种合并方式可以提升程序的可读性，因为程序代码与程序输出的对应关系更紧密。

1.3 注释和空格

1.3.1 注释

注释是用来向代码中添加说明和描述的文字，它们不会被编译器执行，JAVA语言中共有3种注释形式。

**单行注释：**使用 // 开头，注释内容从 // 开始到本行结束。

**多行注释：*使用 / 开头，*/ 结尾，可以注释多行内容。

*JavaDoc 注释：使用 / 开头，/ 结尾，用于生成API文档的特殊注释。可以通过javadoc工具自动生成HTML格式的文档。

JavaDoc常用标签

标签 描述 示例
@author 作者 @author John Doe
@version 版本 @version 1.0
@param 方法参数 @param name 用户名
@return 返回值 @return 计算结果
@throws 异常 @throws IOException 文件读写异常
@see 参考 @see OtherClass
@since 起始版本 @since 1.8

1.3.2 空格

在Java中，空格（包括空格符、制表符、换行符等）被称为空白字符。

编译器只会识别1个空格，而忽略多余空格。

良好的缩进与空格规范

  • 使用空行分隔概念上不同的语句。

  • 保持相同缩进。

  • 在运算符（如=、+、*）前后各加一个空格，使语句更易读。

1.4 错误和警告

1.4.1 语法错误

语法错误是指程序员编写的代码不符合Java编程语言的语法规则，导致编译器无法理解或编译代码的错误。

这类错误会在编译阶段被Java编译器检测出来，因此也称为编译错误。

语法错误特点：

  • 在编译时就被发现
  • 程序无法通过编译，也就无法运行
  • 通常会被IDE（集成开发环境）用红色波浪线标记
  • 修复相对简单，只需要按照语法规则修改代码

常见的语法错误：

  • 缺少分号

  • 括号缺失 / 括号不匹配

  • 关键字拼写错误

  • 变量未声明 / 未初始化

  • 数据类型不匹配

  • 类名和文件名不匹配

  • 缺少引号

  • 方法调用错误，如缺少形参。

1.4.2 逻辑错误

逻辑错误是指程序能够正常运行（没有编译错误和运行时异常），但产生的结果与预期不符的错误。

逻辑错误的特点

  • 程序可以正常运行

  • 没有报错信息

  • 结果与预期不符

  • 是最难发现的错误（通常需要调试才能找到）

常见的逻辑错误

  • 条件判断错误
  • 循环错误
  • 运算符使用错误
  • 数值计算错误
  • 数组索引错误
  • 算法逻辑错误

1.4.3 编译器警告

  • 编译器警告是Java编译器在编译过程中发出的提示信息，表示代码在语法正确的前提下，可能存在潜在问题或不推荐的写法。

  • 警告不会阻止程序编译和运行，但建议开发者关注并修复。

  • 默认情况下，Java 编译器不会打印所有可能的警告。

1.5 计算机和程序概述

1.5.1 开关与计算机的诞生

开关的发明及其衍生的“开/关”逻辑，是现代电子计算机得以诞生的最根本的物理和逻辑基石。

  • 手动开关：控制电灯的通断。
  • 继电器（自动开关）：实现了远程控制和逻辑门电路（与、或、非）。
  • 电子管/真空管（高速电子开关）：实现了每秒数百万次的通断，使ENIAC等电子计算机成为可能。
  • 晶体管（更小、更快的开关）：取代了笨重、发热大的电子管，催生了个人电脑的诞生。
  • 集成电路（集成上亿个微小开关）：将数十亿个晶体管（开关）封装在指甲大小的芯片上，即今天的CPU。

1.5.2 处理器和内存

处理器

定义： 中央处理器，英文简称CPU，是计算机最核心的部件。

核心功能：

  • 解释指令：它不断地从内存中读取软件的指令，比如“计算1+1”、“把这个字显示在屏幕上”。
  • 处理数据：执行这些指令，进行算术运算（加减乘除）、逻辑运算（判断对错）和控制操作。
  • 协调工作：指挥其他硬件（如显卡、硬盘）协同工作。

关键指标：

  • 核心数：多核心处理器可以同时处理多项任务，能更高效地同时运行多个程序。
  • 主频（时钟频率，单位GHz）：频率越高，CPU在单位时间内能执行的指令就越多，速度越快。
  • 缓存：它比内存快得多，用来存储CPU即将用到的数据，以减少去内存“取料”的等待时间。
内存

定义： 内部存储器，英文简称RAM。它是一种临时数据存储器件。

核心功能：

  • 临时存储：当你打开一个程序，它会被从硬盘里读取出来，放到内存中。
  • 高速读写：CPU直接从内存中读取和写入数据，速度远比从硬盘读取要快。这就是为什么程序在内存里才能流畅运行。
  • 断电丢失：内存是“易失性”的。一旦计算机关机或断电，里面存储的所有数据都会立即消失。

关键指标：

  • 容量（单位GB）：相当于料理台的大小。内存越大，能同时顺畅运行的软件就越多。如果内存太小，计算机就会变得卡顿，因为它不得不频繁地去速度慢得多的硬盘里临时“借地方”（使用虚拟内存）。
  • 频率（单位MHz）：决定了内存和CPU之间数据传输速度的快慢。

1.5.3 处理器指令

处理器指令是CPU能直接理解和执行的、最原始的操作命令。

它是软件和硬件之间的桥梁——无论你写的是Python、Java还是C++代码，最终都会被翻译成一条条由0和1组成的处理器指令，交给CPU去执行。

指令的物理本质：电信号

在物理层面，计算机只认识0和1（二进制）。
一条指令实际上是一串特定长度的二进制数（比如 10110000 01100001）。

  • 前面的部分叫操作码，告诉CPU要做什么（例如：加法、读取数据）。
  • 后面的部分叫操作数，告诉CPU操作的对象是什么（例如：数据在哪里，或者具体的数字是多少）。
指令告诉CPU做什么？

CPU能够执行的指令通常分为以下几大类：

  • 算术运算：做加减乘除、比较大小。比如 ADD R1, R2（把R1寄存器和R2寄存器里的数相加）。
  • 逻辑运算：做“与、或、非”等逻辑判断。
  • 数据传送：在内存和CPU之间搬运数据。比如 LOAD（从内存读数据到CPU）和 STORE（把CPU的数据写回内存）。
  • 控制转移：改变程序的执行顺序。比如 JUMP（跳转到另一行代码执行）或 IF...THEN... 这类条件判断。
  • 输入/输出：和键盘、鼠标、硬盘等设备交换数据。
指令的执行过程（指令周期）

一条指令从被CPU接收到执行完毕，通常会经历三个步骤，也就是机器周期：

  1. 取指：CPU从内存中拿到下一条要执行的指令。
  2. 译码：CPU内部的电路分析这条指令是什么意思（是加法？还是读数据？）。
  3. 执行：CPU指挥对应的运算单元完成操作。
常见指令集

  • x86：主要用于英特尔和AMD的电脑/服务器CPU。这种指令集比较复杂，通常称为复杂指令集计算机。

  • ARM：主要用于手机、平板、苹果M系列芯片。这种指令集相对精简，通常称为精简指令集计算机。

  • RISC-V：一种新兴的开源指令集，多用于物联网设备。

1.5.4 编程语言发展

1. 机器语言时代（20世纪40年代）
  • 特征：程序员直接使用二进制代码（0和1）来编写指令。
  • 例子：一条指令可能是 10110000 01100001（代表将某个值存入寄存器）。
  • 缺点：极度繁琐、容易出错、可读性差，且完全依赖于特定的硬件（不同型号的计算机指令集不同）。
2. 汇编语言时代（20世纪50年代初）
  • 特征：为了解决机器语言的难记问题，出现了助记符。汇编语言通过简单的单词（如 MOV, ADD）来代替二进制指令。
  • 例子：MOV AL, 61h（将十六进制数值61移入AL寄存器）。
  • 进步：大大提高了编程效率。
  • 缺点：依然依赖于特定的机器硬件，被称为“低级语言”，移植性很差。
3. 高级语言的萌芽与“编译”时代（20世纪50年代—60年代）

这是编程语言史上的大爆炸时期。人们发明了“编译器”，允许人们用接近数学或英语的方式写代码，由编译器翻译成机器语言。

  • Fortran (1957年)：由IBM开发，是第一个被广泛使用的高级语言。专注于科学计算，至今仍在超算领域活跃。
  • Lisp (1958年)：专为人工智能研究设计，引入了函数式编程和递归，对后世影响深远。
  • Cobol (1959年)：面向商业数据处理，语法接近英语，在银行和政府系统中统治了数十年。
  • Basic (1964年)：设计目标是“简单易学”，让编程在大学生和爱好者中普及开来。
4. 结构化编程与系统编程时代（20世纪70年代）

随着软件规模扩大，“面条式代码”变得难以维护，结构化方法论应运而生。

  • C语言 (1972年)：由丹尼斯·里奇在贝尔实验室设计。C语言既具有高级语言的特性，又能直接操作内存，兼顾了效率与可移植性。它被用来重写Unix操作系统，成为系统编程的霸主，至今仍是编程界的“通用语”。
  • Pascal (1970年)：主要用于教学，强调良好的编程习惯和数据结构。
5. 面向对象编程的兴起（20世纪80年代）

为了应对更复杂的图形界面和软件工程需求，“对象”（将数据和操作封装在一起）的概念开始流行。

  • C++ (1983年)：由Bjarne Stroustrup在C语言基础上引入面向对象机制。C++既有C的高效，又有类的抽象，成为大型系统（如游戏、办公软件、浏览器）的首选。
  • Objective-C (1984年)：同样基于C，但受Smalltalk影响，后来成为苹果（Apple）生态的核心语言。
  • Perl (1987年)：被誉为“管道胶带”，擅长文本处理，在早期互联网 CGI 脚本中非常流行。
6. 互联网时代与脚本语言的爆发（20世纪90年代）

万维网的诞生彻底改变了编程的格局，对快速开发和网页交互的需求剧增。

  • Python (1991年)：由Guido van Rossum创建。注重代码的可读性和简洁性，当时虽未立刻成为主流，但奠定了日后成为“胶水语言”和AI首选语言的基础。
  • Ruby (1995年)：强调“程序员快乐”，后来因Ruby on Rails框架在Web开发中名声大噪。
  • Java (1995年)：由Sun公司推出，口号是“一次编写，到处运行”。它通过Java虚拟机实现了跨平台，迅速占领了企业级后端和安卓开发市场。
  • JavaScript (1995年)：由Brendan Eich在10天内设计完成，最初只是为了在浏览器中做简单的表单验证。但它后来成为了网页交互的核心，随着Node.js的出现，它甚至杀回了服务器端。
  • PHP (1995年)：专为网页开发而生，可以直接嵌入HTML，驱动了无数动态网站（如早期的Facebook和维基百科）。
7. 平台化与托管语言时代（21世纪初）

微软等巨头推出了基于虚拟机的现代框架。

  • C# (2000年)：微软跟随Java的理念，在.NET框架上推出的强类型语言，是Windows桌面和企业开发的主力。
  • Go (2009年)：由Google为解决大规模分布式系统开发效率问题而设计，语法简洁，并发性能极强，成为云计算基础设施（如Docker、Kubernetes）的常用语言。
  • Rust (2010年)：由Mozilla发起，旨在提供C++的性能同时保证内存安全，连续多年被评为“最受喜爱的编程语言”。
8. 当代与未来：AI、数据与全栈（2010年代至今）
  • Python：凭借简单的语法和极其丰富的库（如TensorFlow, PyTorch），Python成为了人工智能、数据科学和机器学习的首选语言。
  • TypeScript (2012年)：微软开发，为JavaScript增加了静态类型，解决了大型前端项目的维护难题，现在已成为前端开发的“新标准”。
  • Kotlin (2011年)：与Java完全互通，但语法更现代，被Google宣布为安卓开发的官方首选语言。
  • Swift (2014年)：苹果推出的用以取代Objective-C的新语言，更安全、更快。
  • 领域特定语言（DSL）与低代码：随着需求多样化，针对特定领域（如硬件描述、AI提示词）的语言也在快速发展。

1.6 集成开发环境（IDE）

集成开发环境（Integrated Development Environment，简称 IDE）是一种为了给程序员提供全面的软件开发设施而设计的应用程序。

1.6.1 常见的 IDE 举例

  • Visual Studio：微软出品，功能极其强大，主要用于 Windows 平台的 C++、C#、.NET 开发，也支持 Python、JavaScript 等。
  • IntelliJ IDEA：JetBrains 公司出品，被认为是 Java 开发最好的 IDE 之一。该公司还为不同语言推出了专门的 IDE（如 PyCharm 用于 Python，WebStorm 用于 JavaScript）。
  • Eclipse：一个开源的、流行的 Java IDE，也通过插件支持其他语言。
  • VS Code：虽然是“代码编辑器”，但通过丰富的插件生态，它已经具备了完整 IDE 的功能，轻量且快速，是目前最流行的开发者工具之一。
  • Xcode：苹果官方提供的 IDE，专门用于 macOS 和 iOS 平台的开发。
