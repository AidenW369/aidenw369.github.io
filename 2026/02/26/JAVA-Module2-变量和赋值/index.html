变量和赋值

2.1 变量和赋值概述

变量（Variable）

变量是用来存储数据的内存空间容器。可以把它想象成一个带有标签的盒子：

  • 有名字：通过变量名来访问这个盒子
  • 有类型：决定盒子里能放什么类型的数据（数字、文字等）
  • 存值：盒子里存放实际的数据

赋值（Assignment）

赋值是将具体的值存储到变量中的操作。使用等号 = 作为赋值运算符。

2.2 变量（整型）

2.2.1 变量声明（int类型）

变量声明是告诉编译器将要创建一个变量，并指定它的类型和名称。

内存分配过程：

  1. 编译时：编译器记录变量名和类型
  2. 运行时：JVM在栈内存中分配4个字节（32位）的空间
  3. 初始值：分配的空间中保留原有的随机值（可能是之前使用过的数据）

2.2.2 赋值语句

赋值语句是使用”=”操作符将一个值存储到变量中。

赋值过程：

  1. 计算右边的表达式（如果有）
  2. 将值转换为变量的类型（如果需要）
  3. 将值写入变量对应的内存位置

2.2.3 变量初始化

变量初始化是在声明变量的同时第一次给它赋值。

重要注意事项

  1. 局部变量必须初始化后才能使用，否则编译错误
  2. 成员变量有默认值（int默认为0）
  3. 每个赋值操作都是将新值写入变量的内存位置
  4. 变量可以多次赋值，但只能初始化一次

2.3 标识符

2.3.1 标识符规则

Java中的标识符是用来给变量、类、方法、包、接口等程序元素命名的名称。

  1. 组成字符限制：
    • 只能包含：字母（A-Z， a-z）、数字（0-9）、美元符号（$）、下划线（_）。
    • 不能包含：空格、@、#等其他特殊符号。
  2. 开头限制：
    • 不能以数字开头。
    • 必须以字母、下划线（_）或美元符号（$）开头。
  3. 保留字限制：
    • 不能是Java关键字或保留字。
    • 例如：int、class、public、static、void、true、false、null等。
  4. 大小写敏感：
    • myVar和myvar是两个完全不同的标识符。

2.3.2 标识符命名风格

1. 帕斯卡命名法 / 大驼峰命名法

  • 定义：每个单词的首字母都大写。
  • 规则：单词单词 -> FirstName， UserLoginController
  • 应用场景：
    • 类名：String， ArrayList， HttpServletRequest
    • 接口名：Runnable， Serializable， List
    • 枚举名：DayOfWeek， Color
    • 注解名：Override， Deprecated

2. 小驼峰命名法

  • 定义：第一个单词首字母小写，后续单词首字母大写。
  • 规则：首单词.后续单词 -> firstName， getUserList
  • 应用场景：
    • 方法名：getName()， setAge(int age)， toString()
    • 局部变量名：userName， totalCount
    • 成员变量名（属性）：private String firstName;

3. 蛇形命名法

  • 定义：所有字母小写，单词之间用下划线（_）分割。
  • 规则：单词_单词 -> first_name， max_value
  • 应用场景：
    • 常量（final static）：这是Java中最常见的使用场景。
    • 数据库字段名：虽然在Java代码中不常见，但在SQL语句或数据库列名中广泛使用。
    • 配置文件中的Key：如 app.config.timeout。

2.4 算数表达式

在Java中，算术表达式是由变量、字面常量、运算符和方法调用按照一定的语法规则组合而成的式子，用于执行数学计算。最终，它会被计算并返回一个数值结果。

2.4.1 算数运算符

2.7 浮点数（双精度）

在Java中，浮点数是指用于表示带有小数部分的数值的数据类型。

2.7.1 Java中的两种浮点数类型

类型 占用字节 占用位数 取值范围（大约） 精度
float 4字节 32位 ±3.4e-38 到 ±3.4e38 单精度（6-7位有效数字）
double 8字节 64位 ±1.7e-308 到 ±1.7e308 双精度（15-16位有效数字）

注意： 在Java中，带小数点的字面量（如 3.14）默认被编译器理解为 double 类型。如果你想使用 float，必须在数字后面加上 f 或 F，否则编译会报错。

2.7.2 使用浮点数的注意事项

1. 精度丢失问题

浮点数不能精确表示所有的小数（特别是十进制小数，如 0.1），因为计算机底层使用二进制，而有些十进制小数转换成二进制是无限循环的。

解决方案： 如果需要精确计算（如货币金额），不要使用 float 或 double，而应该使用 BigDecimal 类。

2. 特殊值

  • Infinity（无穷大）：一个数超出了浮点数的最大值。例如：1.0 / 0.0。
  • NaN（Not a Number，非数字）：表示未定义或无法表示的值。例如：0.0 / 0.0 或负数的平方根。

3. 浮点数比较陷阱

由于精度问题，两个浮点数不应该直接使用 == 进行比较。

2.9 常量

在Java中，常量是指在程序运行期间值不能被修改的变量。

2.9.1 使用final关键字定义常量

被final修饰的变量：

  • 只能被赋值一次
  • 赋值后不能再修改

2.9.2 枚举常量

  • 枚举常量是Java 5引入的一种特殊类型，用于定义一组固定的、预定义的常量集合。

  • 枚举常量通过enum关键字定义，是类型安全的常量表示方式。

2.9.3 Java中的常量主要特点

  • 使用final关键字修饰
  • 通常与static结合使用
  • 命名规范：大写字母+下划线
  • 值不可变
  • 编译时可能被内联优化

2.9.4 使用常量的优势

  1. 可读性：使用有意义的名称替代数字
  2. 可维护性：只需要修改一处就能更新所有引用
  3. 安全性：防止意外修改
  4. 性能优化：编译器可以进行优化
