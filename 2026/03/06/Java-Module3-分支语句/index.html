3.5 逻辑运算符的检测范围

在Java中，逻辑运算符主要用于操作布尔类型（boolean）的数据，即对 true 或 false 进行逻辑运算，最终返回一个布尔值结果。

它们常用于控制流程（如 if、while 语句）中，用来组合多个条件。

3.5.1 逻辑与 （&&） —— 短路与

  • 语法：条件A && 条件B
  • 结果：只有当 两个条件都为true 时，结果才为 true；否则为 false。
  • 特点（短路）：如果条件A为 false，那么整个表达式的结果已经确定为 false，条件B将不会被计算（被短路了）。这可以提高效率并避免潜在的错误。

3.5.2 逻辑或 （||） —— 短路或

  • 语法：条件A || 条件B
  • 结果：只要有一个条件为true 时，结果就为 true；只有两个都为 false 时，结果才为 false。
  • 特点（短路）：如果条件A为 true，那么整个表达式的结果已经确定为 true，条件B将不会被计算。

2.5.3 逻辑非 （!）

  • 语法：!条件
  • 结果：取相反的值。如果条件为 true，则返回 false；如果条件为 false，则返回 true。这是一个一元运算符。

在实际开发中：

  • 绝大多数情况下，使用的是 &&（短路与） 和 ||（短路或）。
  • 利用短路特性可以写出更简洁安全的代码，例如在判断一个对象是否为空之后再获取它的属性：if （obj ！= null && obj.getName（）.equals（“xxx”））。如果 obj 是 null，后半句不会执行，从而避免了空指针异常。

3.7 多条件分支语句

在Java中，多条件分支语句是指根据多个不同的条件，选择执行不同代码块的语句。

执行逻辑：

  • 程序会从上到下依次判断条件。

  • 一旦遇到一个为 true 的条件，就执行对应的代码块，然后整个 if 结构结束（后面的 else if 和 else 都不会再判断）。

  • 如果所有条件都为 false，则执行 else 中的代码（如果存在 else 的话）。

3.10 运算符优先级规则

3.10.1 位运算符（二进制）

在现代计算机中，所有数据都以二进制形式存储，即 0 和 1 两种状态。计算机对二进制数据进行的运算（如加、减、乘、除）被称为位运算，即对二进制数的每一位进行操作的运算。

1. 基本位运算符

运算符 名称 示例 说明
& 按位与 a & b 两个位都为1时，结果为1，否则为0
` ` 按位或 `a
^ 按位异或 a ^ b 两个位相同时为0，不同时为1
~ 按位取反 ~a 0变1，1变0（一元运算符）
<< 左移 a << n 将a的二进制位向左移动n位，右边补0
>> 右移 a >> n 将a的二进制位向右移动n位，左边补符号位
>>> 无符号右移 a >>> n 将a的二进制位向右移动n位，左边补0

3. 注意事项

  • 位运算符只能用于整数类型（byte、short、int、long）
  • 对byte和short进行位运算时，会自动提升为int类型
  • 移位运算如果移动位数超过类型位数，会取模处理
  • 负数的位运算要特别注意符号位

3.10.2 Java中的运算符优先级规则

运算符优先级决定了在表达式中多个运算符共存时，哪个运算先执行。

优先级总表（从高到低）

优先级 运算符类型 运算符 结合性
1 括号/访问 (), [], . 从左到右
2 一元运算符 ++, --, +, -, !, ~**** 从右到左
3 乘除余 *, /, % 从左到右
4 加减 +, - 从左到右
5 移位 <<, >>, >>> 从左到右
6 关系比较 <, >, <=, >=, instanceof 从左到右
7 相等比较 ==, != 从左到右
8 按位与 & 从左到右
9 按位异或 ^ 从左到右
10 按位或 ` `
11 逻辑与 && 从左到右
12 逻辑或 `
13 三元运算符 ? : 从右到左
14 赋值运算符 =, +=, -=, *=, /=, %= 等 从右到左

注意：

  • 圆括号 () 可用于显式改变运算顺序，其优先级最高（相当于手动指定）。
  • 逻辑与（&&）优先级高于逻辑或（||）。
  • 赋值运算符的优先级最低，且从右向左结合，例如 a = b = c 等价于 a = (b = c)。
  • 移位运算符 >>> 是无符号右移，>> 是带符号右移。
