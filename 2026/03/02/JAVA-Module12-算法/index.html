12.1 搜素和算法

12.1.1 线性搜素算法

线性搜索（Linear Search）是最简单的搜索算法，它逐个检查数组或集合中的每个元素，直到找到目标元素或遍历完所有元素。

12.1.2 线性搜素时间

线性搜索的时间复杂度用大O表示法表示：

  • 最好情况 O(1)：目标元素在第一个位置

  • 最坏情况 O(n)：目标元素在最后一个位置或不存在

  • 平均情况 O(n)：目标元素在中间某个位置

12.2 二分查找

二分查找（Binary Search）是一种在有序数组中查找特定元素的高效算法。它的核心思想是分而治之，每次将查找范围缩小一半。

核心原理

  1. 每次比较数组中间元素与目标值
  2. 如果相等，返回索引
  3. 如果中间元素大于目标值，在左半部分继续查找
  4. 如果中间元素小于目标值，在右半部分继续查找
  5. 重复直到找到目标或范围为空

使用Java标准库实现二分查找

确定索引值-向下取整原则

  • 在Java的二分查找实现中，通常通过计算中间索引来确定第一个要比较的元素。
  • 中间索引 mid = (low + high) / 2
  • 整数除法会自动舍弃小数部分（即向下取整）

例如数组 [0, 1, 2, 3, 4, 5]，中间索引 = (0 + 5) / 2 = 2.5，向下取整为2，通常为左边数值。

向下取整：|3.8| = 3 |-2.4| = -3

12.3 大写O符号

  • 大O符号是用于描述算法时间复杂度或空间复杂度的数学表示方法，它表示当输入规模趋近无穷大时，算法执行时间或内存使用的增长率。
  • 大O符号描述的是算法的渐进上界，即算法在最坏情况下的性能表现。
  • 它关注的是算法效率随输入规模增长的变化趋势，而不是具体的执行时间。

12.4 算法分析

最坏时间复杂度

  • 最坏时间复杂度表示算法在最不利的输入情况下所需的运行时间，通常用大O记号（Big O notation）表示。

  • 它给出了算法运行时间的上界，确保算法在任何情况下都不会比这个时间更差。

案例分析1

执行次数计算

  1. 外层循环：执行 N 次
  2. 内层循环：每次外层循环，内层循环执行 N 次
  3. 总迭代次数：N × N = N² 次
  4. 最坏时间复杂度 = O(N²)

案例分析2

每次循环，nVal的值大约减少一半：

  • 第1次后：nVal ≈ N/2
  • 第2次后：nVal ≈ N/4
  • 第3次后：nVal ≈ N/8
  • 第k次后：nVal ≈ N/2^k
  • 循环终止条件：nVal ≤ 0
  • 当 N/2^k ≤ 0 时（实际上是当 N/2^k < 1 时，因为整数除法），循环结束
  • 即：2^k > N，解得：k > log₂N
  • 最坏时间复杂度 = O(log N)
为什么可以忽略底数？

  • 大O记法关心的是增长趋势，而不是具体数值

  • 当 N → ∞ 时：O(log₂N) 和 O(log₁₀N) 的增长曲线形状完全相同，只是垂直方向上的缩放（常数因子）

案例分析3

分析过程：

  1. 循环次数固定：循环执行24次（i < 24），不随输入规模变化

  2. 循环内部操作固定：每次迭代执行常数次比较和赋值操作

  3. 没有嵌套循环：代码结构简单，没有递归或其他复杂结构

  • 最好情况：O(1) - 无论输入如何，都是24次迭代

  • 最坏情况：O(1) - 与最好情况相同

  • 平均情况：O(1) - 执行次数恒定

12.5 排序算法

排序算法是计算机科学中最基础和重要的算法之一。在Java中，排序指的是将一组数据按照特定的顺序（通常是升序或降序）重新排列的过程。

12.5.1 常见的排序算法详解

A. 冒泡排序 （Bubble Sort）

  • 思想：重复遍历数组，依次比较相邻的两个元素，如果顺序错误就交换。每一轮遍历都会把最大（或最小）的元素“浮”到数组末尾。

  • 时间复杂度：O(n²)

  • 空间复杂度：O(1)

  • 稳定性：稳定

  • Java实现要点：双层循环，内层循环比较相邻元素。

B. 选择排序 （Selection Sort）

  • 思想：每一轮从未排序的部分找到最小（或最大）的元素，将其放到已排序序列的末尾。

  • 时间复杂度：O(n²)

  • 空间复杂度：O(1)

  • 稳定性：不稳定（例如：数组 [5, 5, 2]，第一个5会被换到最后）。

  • Java实现要点：记录最小值的索引，遍历完成后进行交换。

C. 插入排序 （Insertion Sort）

  • 思想：将数组分为已排序和未排序两部分，每次从未排序部分取出第一个元素，插入到已排序部分的正确位置（类似打扑克牌时理牌）。

  • 时间复杂度：O(n²) （但对于近乎有序的数据，效率非常高，接近 O(n)）

  • 空间复杂度：O(1)

  • 稳定性：稳定

D. 希尔排序 （Shell Sort）

  • 思想：插入排序的改进版。通过设定一个间隔（gap），将数组分组进行插入排序，然后逐步缩小间隔，直到间隔为1。它可以让元素一次移动较大的距离。

  • 时间复杂度：取决于步长序列，平均 O(n^1.3) 左右

  • 空间复杂度：O(1)

  • 稳定性：不稳定

E. 归并排序 （Merge Sort）

  • 思想：采用分治法（Divide and Conquer）。将数组递归地分成两半分别排序，然后将两个有序的子数组合并成一个完整的有序数组。

  • 时间复杂度：O(n log n) （非常稳定，无论什么情况都是这个复杂度）

  • 空间复杂度：O(n) （需要额外的数组空间来合并）

  • 稳定性：稳定

  • Java实现要点：需要写一个递归的拆分函数和一个合并两个有序数组的函数。

F. 快速排序 （Quick Sort）

  • 思想：也采用分治法。选择一个基准值（pivot），通过一趟扫描将数组分成两部分，使得左边的元素都小于等于基准，右边的都大于等于基准，然后递归地对左右两部分进行排序。

  • 时间复杂度：平均 O(n log n)，最坏情况 O(n²)（当数组已经有序且选基准不当）

  • 空间复杂度：O(log n) （递归调用栈占用的空间）

  • 稳定性：不稳定

  • Java实现要点：核心是 partition（分区）函数，将数组按基准值切分。

G. 堆排序 （Heap Sort）

  • 思想：利用堆这种数据结构。先将数组构建成一个最大堆/最小堆，然后反复将堆顶元素（最大值/最小值）与末尾元素交换，并调整剩余元素为堆。
  • 时间复杂度：O(n log n)
  • 空间复杂度：O(1)
  • 稳定性：不稳定

12.5.2 如何选择排序算法？

  • 数据量很小：插入排序或选择排序简单直接。
  • 数据量很大：快速排序（通常最快）、归并排序（稳定）、堆排序（节省空间）。
  • 数据几乎已经有序：插入排序效率极高。
  • 需要稳定性：归并排序或 TimSort。
  • 在实际工作中：直接用 Arrays.sort() 或 Collections.sort()，Java官方已经针对各种场景做了极致的优化。

12.6 选择排序

  • **基本思想：**每一轮从待排序的数据元素中选出最小（或最大）的一个元素，放在序列的起始位置，直到全部待排序的数据元素排完。

  • 以升序排序为例：

  1. 在未排序序列中找到最小元素，存放到排序序列的起始位置
  2. 从剩余未排序元素中继续寻找最小元素，放到已排序序列的末尾
  3. 重复第二步，直到所有元素均排序完毕

12.7 插入排序

12.7.1 列表判断

1. 已排序列表 (Sorted)

这是最简单的情况。列表中的元素完全遵循升序或降序规则。

  • 判定标准： 对于数组中任意相邻的元素 a[i] 和 a[i+1]，都满足 a[i] <= a[i+1]（升序）或 a[i] >= a[i+1]（降序）。
  • 特征： 逆序对（Inversion Pair）的数量为 0。
  • 检测方法： 遍历一遍数组，检查是否所有相邻元素都符合单调性。时间复杂度为 O(n)。
  • **关于字母的排序：**在Java中，对字母和单词进行排序，核心是基于字符的 Unicode/ASCII 码值 来比较判断，对于升序列表，码值小的排在前面，大的排在后面。

2. 近似排序列表 (Nearly Sorted)

这在实际应用（如日志处理、实时传感器数据）中非常常见。它指的是列表大部分是有序的，只有少数元素偏离了位置。

  • 判定标准： 并没有极其严格的数学定义，但通常满足以下特征之一：
    1. K-排序： 每个元素距离其最终排序后的位置不超过 k 个位置（k 较小）。
    2. 逆序对较少： 逆序对的数量远小于 n(n-1)/4（即无序程度远低于随机状态）。
    3. 移除少数元素即可有序： 只需要移动极少数元素，剩下的序列就是有序的。
  • 特征： 插入排序（Insertion Sort）在这种列表上表现极快，时间复杂度接近 O(n)。
  • 检测方法： 计算“逆序数”或者观察排序算法的交换次数。

3. 无序列表 (Unsorted / Random)

元素随机分布，没有任何明显的排列规律。

  • 判定标准： 元素分布杂乱，单调性频繁切换。
  • 特征： 逆序对的数量通常接近最大可能值的一半，即约为 (n(n-1)) / 4。
  • 检测方法： 如果在检测单调性时，发现违背规则的次数超过了某个阈值（例如 10%），即可认定为无序。

12.7.2 插入排序

基本原理

将数组分为已排序和未排序两部分，每次从未排序部分取出一个元素，插入到已排序部分的正确位置。

算法步骤

  1. 从第一个元素开始，该元素可以认为已经被排序
  2. 取出下一个元素，在已经排序的元素序列中从后向前扫描
  3. 如果该元素（已排序）大于新元素，将该元素移到下一位置
  4. 重复步骤3，直到找到已排序的元素小于或等于新元素的位置
  5. 将新元素插入到该位置后
  6. 重复步骤2~5

执行过程示例

算法复杂度

  • 时间复杂度：

    • 最好情况：O(n) - 数组已经有序
    • 最坏情况：O(n²) - 数组逆序
    • 平均情况：O(n²)

  • 空间复杂度：O(1) - 只需要一个额外变量

  • 稳定性：稳定（相等元素的相对位置不变）

  • 比较总次数（数组已经有序）：n-1

  • 比较总次数（数组完全逆序）：n(n-1) / 2

适用场景

  • 数据量较小（通常n < 1000）
  • 数据基本有序的情况
  • 作为更复杂排序算法的子过程（如希尔排序的基础）

12.8 快速排序

12.8.1 基本概念

  • 快速排序（Quick Sort）是一种高效的排序算法，采用分治法（Divide and Conquer）策略。

  • 通过一趟排序将待排序数据分割成独立的两部分，其中一部分的所有数据都比另一部分小，然后再递归地对这两部分进行排序。

算法步骤详解

  1. 选择基准：从数组中选择一个元素作为”基准”（pivot）
  2. 分区操作：重新排列数组，所有比基准小的元素放在基准前面，比基准大的放在后面
  3. 递归排序：递归地将小于基准的子数组和大于基准的子数组排序

复杂度分析

  • 时间复杂度：
    • 最好情况：O(n log n)
    • 平均情况：O(n log n)
    • 最坏情况：O(n²)（当数组已经有序或逆序时）
  • 空间复杂度：O(log n)（递归调用栈的开销）

分区界定逻辑

  • 枢轴点 (Pivot): 通常从数组中选出的一个特定数值。虽然它可以是随机的，但在基础实现中，通常选择当前区间的第一个元素 arr[low] 或最后一个元素 arr[high]。
  • 低分区 (Low Zone): 所有索引位于 [original_low, pivot_index - 1] 范围内的元素。这些元素的值都小于或等于枢轴点。
  • 高分区 (High Zone): 所有索引位于 [pivot_index + 1, original_high] 范围内的元素。这些元素的值都大于枢轴点。
